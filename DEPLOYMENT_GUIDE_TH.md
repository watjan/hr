## 1. คุณสมบัติและความสามารถระบบ
* **ระบบสารสนเทศทะเบียนประวัติและสถิติกำลังพล (Employee Registry & Dashboard)** เครื่องมือวางแผนบุคลากรที่อัปเดตแบบรีลไทม์
* **ระบบจัดการคำนวณเงินเดือน เงินสมทบประกันสังคมสะสม และออกสลิปเงินเดือน (Payroll & SSO Engine)**
* **ระบบการขออนุมัติการทำงานและบันทึกใบลาอิเล็กทรอนิกส์ (Leave Management)**
* **ระบบคุมเครื่องสารสนเทศหนังสือสำคัญทางการเงินและเช็คขารับ-ขาจ่าย (Cheque Registry)** พร้อมฐานข้อมูลผู้สั่งจ่าย/ซัพพลายเออร์ที่อัปเดตล่าสุด
* **ระบบสำรองข้อมูลประสานข้อมูลอัตโนมัติ (Durable Auto-Sync with Cloud Backups)** เพื่อป้องกันการสูญหายของข้อมูลสำคัญ

## 2. ความต้องการขั้นต่ำของระบบ (System Requirements)
เครื่องเซิร์ฟเวอร์หรือคอมพิวเตอร์ที่ต้องการใช้ติดตั้งระบบนี้จำเป็นต้องมีคุณสมบัติและโปรแกรมดังต่อไปนี้:
* **Node.js** รุ่น v18.0.0 หรือใหม่กว่า (แนะนำ LTS รุ่น v20+)
* **npm** หรือโปรแกรมจัดการแพ็คเกจอย่าง **yarn/pnpm**
* บัญชีผู้ใช้งาน **Google Account** (สำหรับใช้บริการ Firebase ฟรีตลอดชีพภายใต้ข้อตกลงจำกัดปริมาณงาน)

## ขั้นตอนที่ 1: การดาวน์โหลดและเตรียมซอร์สโค้ด
1. ดึงซอร์สโค้ดนี้ออกจาก **Google AI Studio** โดยเข้าไปที่เมนู **Settings** (รูปฟันเฟืองด้านล่างซ้าย)
2. เลือกตัวเลือก **Download ZIP** หรือเชื่อมโยงผ่านปุ่ม **Export to GitHub** เพื่อส่งสิทธิ์ไปยังคลังข้อมูลปลายทางของคุณ
3. แตกไฟล์ ZIP ไปยังโฟลเดอร์สำหรับพัฒนาที่ต้องการในเครื่องของคุณ

## ขั้นตอนที่ 2: การสร้างโครงการฐานข้อมูลคลาวด์ (Firebase Setup)
โปรแกรมชิ้นนี้ได้รับการออกแบบมาให้ใช้ฐานข้อมูลประเภท NoSQL ทรงประสิทธิภาพสูงของ Google ในการเก็บความปลอดภัยของข้อมูล ให้ดำเนินการตามขั้นตอนดังนี้เพื่อสร้างโครงการใหม่ฟรี:

1. เข้าชมเว็บไซต์ [Firebase Console](https://console.firebase.google.com/) แล้วล็อกอินด้วยบัญชี Google ของคุณ
2. คลิกที่ปุ่ม **Add project** (เพิ่มโครงการ) และตั้งชื่อโครงการ เช่น `sapphire-erp-system` จากนั้นกดดำเนินการต่อจนเสร็จสิ้น
3. บนหน้าแผงควบคุมหลัก ให้มองหาสนับสนุนของแอปโดยการเพิ่มระบบเว็บ **Web Application** (คลิกไอคอนรูปวงเล็บสามเหลี่ยม `< />` บนหน้าหลัก) 
4. ตั้งชื่อเว็บแอปของท่านแล้วกดลงทะเบียน (Register App) ระบบจะแสดงรหัสการเชื่อมโยง Firebase Software Development Kit (SDK Config) ซึ่งจะมีลักษณะดังต่อไปนี้:
   ```json
   {
     "apiKey": "AIzaSy...",
     "authDomain": "your-project.firebaseapp.com",
     "projectId": "your-project",
     "storageBucket": "your-project.firebasestorage.app",
     "messagingSenderId": "YourSenderId",
     "appId": "1:yourAppId"
   }
   ```
5. บันทึกข้อมูลค่าเหล่านี้เก็บเอาไว้ ท่านสามารถนำรหัสชุดนี้ไปใส่ไว้ในไฟล์ **`firebase-applet-config.json`** ในโฟลเดอร์หลักของโปรแกรมได้โดยตรง หรือนำมาใช้ตั้งค่าในไฟล์ `.env` (ตัวแปรสภาพแวดล้อม)

### การสร้างอินสแตนซ์ฐานข้อมูล (Firestore Database)
1. ในแถบเมนูด้านซ้ายของ Firebase Console ไปที่หัวข้อ **Build** -> **Firestore Database**
2. คลิกที่ปุ่ม **Create database** 
3. เลือกภูมิภาคโฮสติ้งเซิร์ฟเวอร์ (แนะนำให้เลือก `asia-southeast1` สิงคโปร์ เพื่อลดอาการดีเลย์และความเร็วสูงสุดในไทย)
4. เลือกรันในระบบความปลอดภัยแบบเริ่มต้น (Test Mode หรือ Production Mode ก็ได้) และกดตั้งค่าให้เสร็จสิ้น

## ขั้นตอนที่ 3: การตั้งค่าและการใช้งานตัวเชื่อมโยงระบบติดตั้งข้อมูล (Database Seeding)
ทางทีมงานได้เตรียมไฟล์ติดตั้งระบบอัตโนมัติพร้อมชุดฐานข้อมูลเริ่มต้นที่เป็นระเบียบเอาไว้ให้ในไฟล์ `src/firebase-seeder.js` เพื่อความสะดวกสูงสุดในการเริ่มต้นระบบใหม่

1. ปรับแต่งข้อมูลเชื่อมต่อฐานข้อมูลในเครื่องเป้าหมาย โดยสร้างไฟล์ `.env` ที่โฟลเดอร์หลักตามตัวอย่างต้นแบบดังนี้:
   ```env
   VITE_FIREBASE_API_KEY="ค่าจากขั้นตอนที่ 2"
   VITE_FIREBASE_AUTH_DOMAIN="ค่าจากขั้นตอนที่ 2"
   VITE_FIREBASE_PROJECT_ID="ค่าจากขั้นตอนที่ 2"
   VITE_FIREBASE_STORAGE_BUCKET="ค่าจากขั้นตอนที่ 2"
   VITE_FIREBASE_MESSAGING_SENDER_ID="ค่าจากขั้นตอนที่ 2"
   VITE_FIREBASE_APP_ID="ค่าจากขั้นตอนที่ 2"
   ```
2. ติดตั้งโมดูลและสคริปต์ที่เกี่ยวข้องจากไฟล์บันทึก โดยการรันตรรกะคำสั่ง:
   ```bash
   npm install
   ```
3. สั่งให้ระบบทำการรันตารางการจัดเก็บและนำเข้าฐานข้อมูลค่าเริ่มต้น (เช่น สารสนเทศบริษัท, แผนกประวัติ, รายงานคำนวณเงินเดือนของพนักงานอัฟเดทปัจจุบัน, สถิติกำลังพล และสิทธิ์เข้าทำหน้าที่ของผู้ใช้) ได้ทันทีผ่านเครื่องมือเขียนชุดคำสั่ง:
   ```bash
   node src/firebase-seeder.js
   ```
4. หากสำเร็จ! ท่านจะเห็นข้อความแจ้งว่าการติดตั้งเสร็จสมบูรณ์และพร้อมทำงานร่วมกับฐานข้อมูลใหม่ทันที

## ขั้นตอนที่ 4: การรันและทดสอบระบบในเครื่องพัฒนาภายใน (Local Development)
ตรวจสอบให้แน่ใจว่าติดตั้งไลบรารีทั้งหมดเรียบร้อยแล้ว จากนั้นสามารถรันระบบเซิร์ฟเวอร์ทดสอบในเครื่องได้ด้วยคำสั่งเดียว:

```bash
npm run dev
```

หน้าเว็บแอปเวอร์ชันเดสก์ท็อปและโมบายล์จะแสดงขึ้นที่พอร์ตอินเตอร์เน็ตวงกว้างประชารัฐของคุณที่ลิงก์ชั่วคราว: `http://localhost:3000`

## ขั้นตอนที่ 5: การพอร์ตและการนำระบบขึ้นเผยแพร่ใช้งานจริง (Production Deployment)
แอปพลิเคชันระบบสารสนเทศ Sapphire ERP ชิ้นนี้เป็น Single Page Application (SPA) ที่พัฒนาผ่านแพลตฟอร์ม React และ Vite ซึ่งไม่มีความจำเป็นต้องใช้การจัดการฝั่ง Backend เพิ่มเติม ( Server-less SPA ) เมื่อต้องการแพ็คและส่งขึ้นโดเมนเพื่อทำเป็นโปรแกรมที่เข้าได้จากทั่วโลก ให้ทำตามขั้นตอนนี้:

### 1. นำชุดไฟล์ประกอบรวมกันเป็น Static Asset:
```bash
npm run build
```
ระบบจะรันการตรวจสอบประเภท TypeScript (Type Check) และย่นขนาดไฟล์ CSS และ Javascript ทั้งหมดไปเก็บลงในไดเรกทอรีโฟลเดอร์ชื่อ **`dist/`**

### 2. ตัวเลือกบริการโฮสติ้งฟรีประสิทธิภาพสูงที่แนะนำ:
คุณสามารถลากและวางโฟลเดอร์ `dist` นี้ไปใช้ส่งเผยแพร่บนระบบต่อไปนี้ได้เลย:
* **Firebase Hosting** (แนะนำที่สุด ทำความปลอดภัย HTTPS ติดตั้งเว็บร่วมกับ Firestore ได้ในไอดีเดียวฟรี)
* **Vercel** หรือ **Netlify** (เชื่อมกับคลังข้อมูล Github เพื่อทำแผงควบคุมระบบคอยสร้างและอัฟเดทเนื้อหาอัตโนมัติ)
* **Cloudflare Pages** (โฮสติงไร้ขีดจำกัดความเร็วทั่วไทย)

## 6. กฎความปลอดภัยของฐานข้อมูล (Security Rules)
เพื่อให้มั่นใจว่าระบบไม่มีความเสี่ยงต่อการแรนซัมแวร์หรือโดนดึงข้อมูลส่วนตัวจากการเชื่อมต่อ ให้ตั้งค่าความปลอดภัยของ Cloud Firestore โดยนำกฎความปลอดภัยในบรรทัดเหล่านี้จากในโครงสร้างของระบบไปวางในแถบเมนู **Rules** บน Firebase Console ของโครงการคุณ:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /sapphire_hr/{document} {
      allow read, write: if true; // แนะนำสำหรับการใช้งานเบื้องต้น หรือปรับแต่งให้อนุมัติเฉพาะผ่าน UUID หากมีการทำระบบ Firebase Authentication ขั้นสูง
    }
  }
}
```
